# Mousehole v0.5.0 is here!

Hi all,

Another update! Changes for efficiency, consistency, and polish are in store.

## Highlights

- Mousehole now simply **updates MAM on every interval** instead of trying to
  detect when the change is needed. The old change-detection logic (and
  `MOUSEHOLE_STALE_RESPONSE_SECONDS`) is gone. Now there are fewer moving parts.
- **Action needed**: the `MOUSEHOLE_CHECK_INTERVAL_SECONDS` env var is renamed
  to `MOUSEHOLE_UPDATE_INTERVAL_SECONDS`. If you set the old name, rename it
  (otherwise the 5-minute default applies). This is part of a larger effort to
  make the naming more consistent.
- On the frontend, **more errors are surfaced if you cannot reach the
  backend**. You now get the actual error and a Retry button instead of an
  endless loading circle or stalled timer.
- The Unraid guide has been overhauled for clarity and the associated template
  updated to match the latest version of Mousehole. Thanks to
  [@chodeus](https://github.com/chodeus).
- Errors that can be attributed to misconfigurations now have **actionable**
  messages with the name of the offending value and the setting to change.
- Live dashboard updates now use **Server-Sent Events** instead of WebSockets.
  SSE is just plain HTTP, so reverse proxies need no special upgrade
  configuration.
- Under the hood, the server moved to [Hono](https://hono.dev) and the frontend
  to [Vite](https://vite.dev) for ergonomics and to remove dependence on our
  runtime platform.
- On the backend, **API endpoints are renamed** for consistency and clarity:
  the endpoints are now `PUT /cookie` (change your cookie value),
  `POST /updates` (trigger an update), and `GET /state` (read the current
  state).

## Upgrading

For most Docker users: pull the new image. See the
[changelog](https://github.com/t-mart/mousehole/blob/master/CHANGELOG.md) for
the full list of changes.

*Try it out when you can and give me feedback!*
